# MVP сравнения заявки и КП

Бюджетная версия под задачу Кирилла: локальный веб-сервис принимает заявку/ТЗ и КП поставщиков, сопоставляет позиции, дает проверить спорные строки и формирует итоговую Excel-сводку.

## Что уже поддержано

- Заявка/ТЗ в `.xlsx` с колонками описания, единицы измерения и количества.
- КП в `.xlsx`, если есть табличные колонки `Товар`, `Кол-во`, `Цена`, `Сумма`.
- КП в `.xlsx` в формате заявки, где цена и срок указаны рядом с позициями.
- Текстовые PDF-счета, где текст извлекается из PDF, а не является чистым сканом.
- Итоговый Excel:
  - слева позиции заявки;
  - справа отдельные блоки поставщиков;
  - прочерк, если позиции у поставщика нет;
  - минимальная цена подсвечивается зеленым;
  - максимальная цена подсвечивается красным;
  - совпадение количества помечается зеленой `✓`;
  - спорные совпадения и разные бренды подсвечиваются;
  - несопоставленные позиции КП выводятся внизу.
- Промежуточный файл проверки: пользователь может исправить колонку `Позиция заявки (исправить при необходимости)` и затем пересобрать финальный отчет с `--review-in`.
- Простой веб-интерфейс:
  - загрузка заявки и 2+ КП;
  - экран ручной проверки спорных строк;
  - скачивание итогового Excel.

## Ограничения MVP

- Старые `.xls` в текущем окружении не читаются без `xlrd`, Excel COM или LibreOffice. Практичный путь: пересохранить такие файлы в `.xlsx` или PDF.
- Сканированные PDF/фото требуют OCR и не входят в базовый объем.
- ИИ/DeepSeek опционален: без ключа сервис работает алгоритмически, с ключом отправляет спорные и несопоставленные строки на дополнительную проверку.

## DeepSeek

Локально ключ можно положить в файл `.env` рядом с кодом:

```text
DEEPSEEK_API_KEY=ваш_ключ
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_AI_SCOPE=all
DEEPSEEK_MAX_AI_ROWS=300
DEEPSEEK_BATCH_SIZE=20
```

Если ключ взят не в официальном DeepSeek, а в совместимом сервисе-шлюзе, нужно также указать его адрес. Например для DotPoin:

```text
DEEPSEEK_API_BASE=https://llms.dotpoin.com/v1
```

На Render эти же значения нужно добавить в `Environment Variables`. Ключ нельзя коммитить в GitHub.

## Запуск веб-интерфейса

```powershell
python web_app.py
```

Или двойным кликом:

```text
run.bat
```

После запуска открыть:

```text
http://127.0.0.1:8000
```

## Запуск через CLI

```powershell
python compare_tz_kp.py `
  --request "extracted\Заявка.xlsx" `
  --offers "extracted\Петрович.xlsx" "extracted\1350.pdf" "extracted\Евростройгрупп.xlsx" `
  --review "outputs\review.xlsx" `
  --out "outputs\summary.xlsx"
```

Если файл проверки исправлен вручную:

```powershell
python compare_tz_kp.py `
  --request "extracted\Заявка.xlsx" `
  --offers "extracted\Петрович.xlsx" "extracted\1350.pdf" "extracted\Евростройгрупп.xlsx" `
  --review-in "outputs\review.xlsx" `
  --out "outputs\summary_checked.xlsx"
```

## Реальная команда для текущих файлов

Имена файлов в архиве пришли с битой кодировкой, поэтому проще выбирать их через автодополнение PowerShell или переименовать в понятные:

- `zayavka.xlsx`
- `petrovich.xlsx`
- `avangard.pdf`
- `eurostroygroup.xlsx`

После переименования команда выше будет работать без длинных нечитаемых имен.
